# Career Direction

## 현재 방향
손기석은 **백엔드 개발을 중심으로 AI 응용을 확장하고, 데이터/플랫폼 역량을 기반으로 쌓으며, 보안 도메인을 차별화 포인트로 활용하는 엔지니어**를 지향합니다.

이 방향은 단순히 최신 키워드를 따라가기 위한 선택이 아니라, 지금까지의 경험과 앞으로의 시장가치를 함께 고려한 결과입니다.

---

## 왜 이 방향을 선택했는가

### 1. 백엔드는 현재 가장 강한 기반이다
지금까지의 실무 경험에서 가장 강한 축은 Java/Spring 기반 백엔드 개발, DB 연계, API 설계, 운영 반영입니다.
따라서 커리어의 중심은 계속 백엔드에 두는 것이 가장 자연스럽고 현실적입니다.

### 2. AI는 서비스 응용 관점에서 확장성이 크다
최근에는 LLM 챗봇 PoC, Text2SQL, 문서 기반 AI 서비스 구조 검토 등 AI 응용서비스와 연결되는 경험을 쌓아왔습니다.
따라서 연구형 AI보다, **서비스에 붙는 AI 기능을 구현하는 엔지니어** 방향이 더 적합합니다.

### 3. 데이터/플랫폼 역량은 오래 남는 자산이다
AI 유행과 별개로, 데이터 흐름, 검색, 배포, 운영, 플랫폼 구조를 이해하는 능력은 장기적으로 남는 역량입니다.
그래서 백엔드 위에 데이터/플랫폼 역량을 계속 쌓는 방향을 함께 가져가고자 합니다.

### 4. 보안 경력은 차별화 포인트가 된다
보안 경력과 자격증, 그리고 통제/감사/운영 맥락을 이해한 경험은 일반 백엔드 개발자와 다른 강점입니다.
따라서 보안은 버릴 과거 경력이 아니라, 앞으로도 차별화 포인트로 활용할 수 있는 자산입니다.

---

## 커리어 방향 3축

### 1. 메인 전략
**백엔드 + AI 응용 확장형**
- Java/Spring 또는 Python/FastAPI 기반 백엔드
- RAG, Text2SQL, Agent, 문서 기반 질의응답
- AI 기능을 서비스에 연결하는 구조 설계와 구현

### 2. 기반 전략
**백엔드 + 데이터/플랫폼 확장형**
- 데이터 흐름 설계
- 검색/배포/운영 구조
- Docker, CI/CD, 로그, API 안정성
- 플랫폼형 서비스 백엔드 역량 축적

### 3. 차별화 전략
**보안 + 백엔드 + 플랫폼 특화형**
- 감사 로그
- 권한/접근통제
- 보안 데이터 처리
- 로그/통제/감사 중심 서비스 구조 이해

---

## 앞으로 만들고 싶은 엔지니어 이미지
- 단순 기능 구현자보다 서비스 구조를 이해하는 엔지니어
- AI를 연구가 아니라 서비스 기능으로 연결할 수 있는 엔지니어
- 운영 가능한 구조를 만들 수 있는 엔지니어
- 필요할 때 보안과 통제 관점까지 같이 볼 수 있는 엔지니어
