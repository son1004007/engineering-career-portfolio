# Working Rules

이 문서는 이 저장소를 손기석님, 다른 ChatGPT 대화, Codex가 이어서 사용할 때 지켜야 할 기본 규칙입니다.

## 1. 저장소 목적을 벗어나지 않는다
이 저장소는 아래 4가지를 위한 저장소입니다.
- 포트폴리오
- 이력서/경력기술서 관리
- 채용공고 분석
- 회사 공유 가능한 전략/산출물 문서화

직접 관련 없는 개인 메모는 넣지 않습니다.

## 2. 문서는 목적별로 분리한다
같은 내용이라도 목적이 다르면 별도 문서로 분리합니다.
예:
- 포트폴리오용 설명
- 회사 공유용 설명
- 면접 답변용 설명
- 채용공고 분석용 설명

## 3. 파일명은 사람이 바로 이해할 수 있게 작성한다
- 영어 파일명 사용
- 날짜가 필요한 문서는 `YYYY-MM-` 형식 사용
- 공고 문서는 회사명과 포지션이 드러나게 작성

## 4. 새 채용공고가 들어오면 먼저 개별 문서를 만든다
순서는 항상 아래와 같습니다.
1. `03_job-postings/postings/` 아래 개별 공고 md 생성
2. 점수화 및 판정 기록
3. `03_job-postings/overall-comparison-summary.md` 갱신
4. 필요한 경우 맞춤 이력서 버전 연결

## 5. 점수 기준은 고정한다
채용공고 평가는 아래 10개 항목으로 고정합니다.
1. 연봉 상승 가능성
2. 업무 선명도
3. 기술 성장성
4. 현재 경력 연결성
5. 시장가치 상승 가능성
6. 업무강도 대비 보상
7. 장기 커리어 적합도
8. 회사/조직 안정성
9. 기술 깊이
10. 개인 선호도

각 5점 만점, 총점 50점입니다.

## 6. 현재 커리어 방향은 임의로 바꾸지 않는다
현재 기준 방향은 아래와 같습니다.
- 메인 전략: 백엔드 + AI 응용 확장형
- 기반 전략: 백엔드 + 데이터/플랫폼 확장형
- 차별화 전략: 보안 + 백엔드 + 플랫폼 특화형

새로운 방향 제안은 가능하지만, 기존 방향을 덮어쓰지 말고 비교안으로 추가합니다.

## 7. 프로젝트 문서는 실행 가능성을 우선한다
프로젝트 문서는 설명만 두지 않습니다.
가능하면 아래 항목을 같이 둡니다.
- README
- 구조 설명
- 실행 방법
- 기술 스택
- 화면/흐름 설명

## 8. handoff를 위해 상태 문서를 최신으로 유지한다
새 작업을 할 때마다 `00_meta/current-status.md` 를 갱신합니다.
필수 반영 항목:
- 최근 완료한 작업
- 다음 할 일
- 현재 상위 우선순위 공고
- 저장소 구조 변경 사항

## 9. 다른 대화/도구가 이어받을 때는 먼저 읽을 문서가 있다
이어받는 도구/대화는 아래 순서로 읽습니다.
1. `README.md`
2. `00_meta/project-context.md`
3. `00_meta/current-status.md`
4. `00_meta/handoff-guide.md`

## 10. 문서는 짧고 분명하게 쓴다
- 제목만 보고 문서 목적이 드러나게 작성
- 불필요하게 장황한 설명보다 구조화된 설명을 우선
- 복붙해 다른 곳에 재사용 가능한 문장을 선호
