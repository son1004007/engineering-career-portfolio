# Project Context

## 저장소 목적
이 저장소는 손기석님의 엔지니어 커리어를 외부에 설명하고, 동시에 실제 이직 준비와 산출물 관리를 지속하기 위한 개인 포트폴리오 저장소입니다.

이 저장소는 아래 4가지 목적을 동시에 만족해야 합니다.
1. 면접관과 채용 담당자가 보기 쉬워야 한다.
2. 손기석님이 직접 이력서/공고/프로젝트를 관리하기 쉬워야 한다.
3. 회사 내부 공유용 문서로도 일부 활용 가능해야 한다.
4. 나중에 다른 ChatGPT 대화나 Codex가 이어받아도 바로 작업할 수 있어야 한다.

---

## 사용자 핵심 배경
- 이름: 손기석
- 직무 축: 백엔드 개발자 / AI 응용서비스 개발자 / 보안 경력 보유
- 핵심 기술: Java, Spring Boot, Python, FastAPI, Oracle, PostgreSQL, SQL, Linux, Docker, Jenkins, Git
- 확장 관심사: RAG, Text2SQL, LangChain, LangGraph, Agent, GraphRAG, LLMOps
- 차별화 요소: 정보보안 경력, 보안 자격증, 운영/배포/개발환경 구성 경험

---

## 현재 메인 커리어 방향
### 1. 메인 전략
백엔드 + AI 응용 확장형

### 2. 기반 전략
백엔드 + 데이터/플랫폼 확장형

### 3. 차별화 전략
보안 + 백엔드 + 플랫폼 특화형

---

## 이 저장소에서 다뤄야 할 핵심 문서 영역
### 1. 프로필
- 내가 어떤 엔지니어인지 요약
- 핵심 강점과 방향 정리

### 2. 이력서
- 포지션별 이력서 버전 분리
- 백엔드형 / 백엔드+AI형 / 보안+백엔드형

### 3. 채용공고
- 개별 공고 분석
- 누적 비교표
- 점수 기준
- 지원 우선순위

### 4. 프로젝트
- 실제 포트폴리오 산출물
- AI 응용 PoC
- 운영형 백엔드 템플릿
- 보안 차별화 기능 예제

### 5. 회사 공유용 문서
- 올해 사업 유형
- 개발 역할 정의
- 공통 산출물 전략

### 6. 면접 준비
- 자주 나오는 질문
- 회사별 메모

---

## 작업 시 가장 중요한 원칙
- 문서는 다른 사람이 읽어도 바로 이해되게 작성한다.
- 설명 문서와 실행/산출물 문서를 분리한다.
- 새 대화나 다른 도구가 이어받을 수 있게 현재 상태 문서를 항상 최신으로 유지한다.
- 저장소가 커질수록 폴더/문서 이름은 더 명확하게 유지한다.
