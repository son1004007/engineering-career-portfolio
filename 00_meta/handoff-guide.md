# Handoff Guide

이 문서는 다른 ChatGPT 대화나 Codex가 이 저장소 작업을 이어받을 때 참고하는 안내서입니다.

## 먼저 읽을 문서 순서
1. `README.md`
2. `00_meta/project-context.md`
3. `00_meta/current-status.md`
4. `00_meta/working-rules.md`

이 4개를 먼저 읽으면 저장소 목적, 현재 상태, 작업 규칙을 빠르게 이해할 수 있습니다.

---

## 이 저장소에서 가장 중요한 사실
- 손기석님은 백엔드 개발자이면서 AI 응용서비스, 데이터/플랫폼, 보안 경력을 함께 가진 엔지니어입니다.
- 저장소의 목적은 단순 문서 보관이 아니라 **포트폴리오 + 이직 운영 + 회사 공유 문서**를 동시에 관리하는 것입니다.
- 향후 작업은 반드시 **다른 사람이 읽기 쉬운 구조**를 우선해야 합니다.

---

## 현재 기준 커리어 방향
### 메인 전략
백엔드 + AI 응용 확장형

### 기반 전략
백엔드 + 데이터/플랫폼 확장형

### 차별화 전략
보안 + 백엔드 + 플랫폼 특화형

따라서 새로운 문서를 작성할 때도 이 3축 중 어디에 속하는지 먼저 판단해야 합니다.

---

## 새 작업을 시작할 때 해야 할 일
### 1. 현재 상태 확인
`00_meta/current-status.md` 에서 최근 완료 작업과 다음 할 일을 확인합니다.

### 2. 기존 자산 확인
기존 `career-playbook` 저장소에 이미 정리된 자산이 있을 수 있으므로, 새로 쓰기 전에 중복 여부를 확인합니다.

### 3. 작업 목적 분류
아래 중 무엇인지 먼저 정합니다.
- 프로필 문서
- 이력서 문서
- 채용공고 분석
- 프로젝트 포트폴리오
- 회사 공유용 문서
- 면접 준비 문서

### 4. 문서 위치 결정
목적에 맞는 폴더에 둡니다.

---

## 채용공고 분석 작업 시 규칙
- 개별 공고 문서를 먼저 만든다.
- 점수 기준은 10개 항목 / 총점 50점 고정이다.
- 개별 공고 작성 후, 전체 비교 요약 문서를 반드시 갱신한다.
- placeholder 문서와 정식 분석 문서를 구분한다.

---

## 프로젝트 문서 작업 시 규칙
- 포트폴리오용 문서는 면접관이 이해하기 쉽게 작성한다.
- 가능한 경우 README, 구조, 실행 방법, 사용 기술, 문제/해결/결과를 함께 적는다.
- 단순 아이디어 메모보다 실행 가능한 구조 문서를 우선한다.

---

## 이력서/포트폴리오 작업 시 규칙
- 하나의 버전으로 모든 공고를 대응하지 않는다.
- 최소 3축을 유지한다.
  1. 백엔드형
  2. 백엔드 + AI 응용형
  3. 보안 + 백엔드형
- 면접관/채용담당자가 빨리 이해할 수 있게 문장과 구조를 단순화한다.

---

## 현재 가장 먼저 옮겨와야 할 기존 자산
기존 저장소 `career-playbook` 에서 아래 문서들을 우선 이전하거나 재작성하는 것이 좋다.
- 채용공고 전체 비교 요약
- 상위권 개별 공고 분석 문서
- 단계별 실행 로드맵
- 커리어 방향 3안 비교표

---

## 작업을 마칠 때 반드시 해야 할 일
- `00_meta/current-status.md` 갱신
- 새로 만든 핵심 문서를 README 경로 구조에 맞게 반영할지 검토
- 이후 다른 도구/대화가 이어받아도 이해 가능한지 확인
