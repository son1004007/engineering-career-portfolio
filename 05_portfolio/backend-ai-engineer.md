# Backend + AI Application Engineer

## 포지셔닝
손기석은 **백엔드 개발을 기반으로 AI 응용서비스를 구현할 수 있는 엔지니어**입니다.

핵심은 단순히 AI 모델을 아는 것이 아니라,
**사용자가 실제로 쓸 수 있는 서비스 흐름 안에 AI 기능을 연결하고 운영 가능한 구조로 정리하는 것**에 있습니다.

---

## 내가 해결하려는 문제
현업에서 AI는 종종 데모나 실험 수준에서 멈춥니다.
하지만 실제 서비스에서는 아래 문제가 함께 해결되어야 합니다.

- 기존 시스템과 어떻게 연결할 것인가
- 사용자 요청과 응답 흐름을 어떻게 설계할 것인가
- 정형/비정형 데이터를 어떻게 연결할 것인가
- 운영 가능한 형태로 어떻게 배포하고 관리할 것인가
- 비용, 품질, 속도 사이의 균형을 어떻게 맞출 것인가

손기석은 이런 문제를 **백엔드 관점에서 AI 기능을 서비스로 연결하는 방식**으로 해결하는 엔지니어를 지향합니다.

---

## 내가 잘하는 일
### 1. 백엔드 시스템 구현
- Java/Spring Boot 기반 웹 애플리케이션 및 API 개발
- Python/FastAPI 기반 API 서버 구현
- DB 연계 및 데이터 흐름 구현

### 2. AI 응용서비스 구조 설계
- LLM 챗봇 PoC
- Text2SQL 구조 구현
- 문서 기반 AI 서비스 구조 검토
- AI 기능을 실제 서비스 플로우에 연결하는 구조 이해

### 3. 운영 가능한 구조 정리
- Linux, Docker, Jenkins, Git 기반 실행 환경 경험
- 개발환경, 실행 가이드, 협업 구조 정리 경험
- 운영 반영과 장애 대응 경험

### 4. 도메인과 통제 관점 이해
- 정보보안 경력 기반의 통제/감사/운영 맥락 이해
- 데이터 분석 결과를 사용자 서비스로 연결하는 경험

---

## 적합한 포지션
손기석은 아래와 같은 포지션에 가장 잘 맞습니다.

### 1. 백엔드 + AI 응용 확장형 포지션
- AI 기능이 붙는 백엔드 서비스 개발
- RAG, Text2SQL, Agent 기반 서비스 구현
- FastAPI + Spring 혼합형 서비스 구조

### 2. 제품형 백엔드 강화 포지션
- 데이터 기반 서비스 백엔드 개발
- 운영형 플랫폼/API 개발
- 서비스 구조 고도화 및 기능 개선

### 3. 보안/통제 기능이 포함된 백엔드 포지션
- 감사로그, 권한, 통제 기능이 필요한 서비스
- 보안/컴플라이언스 도메인과 연결된 백엔드 개발

---

## 차별화 포인트
### 일반적인 백엔드 개발자와의 차이
- AI 기능을 서비스 관점에서 연결해 본 경험이 있음
- Python/FastAPI와 Java/Spring 두 축을 모두 활용 가능
- 운영/배포/실행 환경까지 고려한 개발 경험이 있음
- 정보보안 경력으로 인해 통제/감사/보안 맥락까지 이해 가능

---

## 한 줄 소개
손기석은 **Java/Spring 기반 백엔드 경험 위에 Python/FastAPI와 LLM 응용서비스 구현 경험을 더하고, 운영·보안 관점까지 함께 볼 수 있는 실무형 엔지니어**입니다.
